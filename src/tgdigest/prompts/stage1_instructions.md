ЭТАП 1 — выявление значимых сообщений.

На входе фрагмент переписки. Каждое сообщение помечено как `#<id>`, далее время,
автор и текст. Строки `--- тред N ---` отделяют обсуждения.

Проанализируй фрагмент и верни СТРОГО один JSON-объект следующего вида:

{
  "events": [
    {
      "message_id": <число — #id ключевого сообщения события>,
      "importance_type": "<один из типов ниже>",
      "summary": "<суть события в 1–2 предложениях, по делу>",
      "reason": "<почему это важно для пользователя>",
      "confidence": <число от 0 до 1 — насколько ты уверен в важности>,
      "related_message_ids": [<#id связанных сообщений, если есть>]
    }
  ]
}

Допустимые значения importance_type:
news, decision, agreement, task, assignment, deadline, warning, instruction,
document, link, open_question, plan_change, attention, recurring_topic, other.

Требования:
- Используй ТОЛЬКО `#id`, которые реально присутствуют во входных данных.
- Одно событие может объединять несколько сообщений через related_message_ids.
- Не дублируй одно и то же событие несколько раз.
- Если значимого во фрагменте нет, верни {"events": []}.
- Ответ — только JSON, без пояснений, без markdown-обрамления.
