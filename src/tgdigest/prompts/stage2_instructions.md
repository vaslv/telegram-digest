ЭТАП 2 — формирование дайджеста.

На входе список значимых событий (тип, краткое описание, причина, ссылки на
сообщения) и анализируемый период. Верни СТРОГО один JSON-объект:

{
  "summary": "<2–4 предложения: общее резюме периода по существу>",
  "key_events": ["<ключевое событие/решение/договорённость>", "..."],
  "attention": ["<что требует внимания или действия пользователя>", "..."],
  "open_questions": ["<заданные, но оставшиеся без ответа вопросы>", "..."],
  "links": ["<полезные внешние ссылки и упомянутые документы>", "..."],
  "conclusion": "<1–2 предложения: итоговый вывод>"
}

Требования:
- Пиши конкретно и кратко, каждый пункт — законченная мысль.
- Объединяй похожие события, не повторяйся.
- Если для какого-то раздела нет содержания — оставь пустой список или пустую
  строку. Не придумывай наполнение.
- В `links` указывай только реально упомянутые в событиях ссылки/документы.
- Ответ — только JSON, без пояснений, без markdown-обрамления.
